命令行以及git初识2

发表于 2017-06-30 | 分类于 技术

git-02

分支操作

  • 创建分支
    • git branch 分支名称
  • 切换分支
    • git checkout 分支名称
  • 创建并切换分支名称
    • git checkout -b 分支名称
  • 删除本地分支
    • git branch -d 分支名称(如果没有被合并,无法被删除)
    • git branch -D 分支名称(强制删除)
  • 查看本地分支
    • git branch
  • 查看远程分支
    • git branch -r
  • 查看所有分支
    • git branch a
  • 删除远程分支
    • git push 远程仓库地址 : 远程分支名称
    • git push 远程仓库地址 – delete 远程分支名称
  • 保存当前状态
    • git stash
  • 回复当前状态
    • git stash pop

远程登录

  • telnet 一种不加密的通信协议
  • ssh 一种协议, 用来实现两台计算机之间的通信(具体就是实现远程登录)
    • 使用密码登录
      • 首先客户端向服务器发送一个请求
      • 服务器接受到请求之后向客户端发送自己的公钥
      • 客户端接受到公钥之后存储到指定目录(C:\ Users\ admin.ssh)
      • 客户端输入密码然后用公钥加密,再次发送请求
      • 服务器接收到密文之后,使用自己的私钥解密然后验证通过,从而允许服务端登录
    • 使用公钥和私钥登录
      • 客户端先生成自己的公钥和私钥(ssh-keygen -t rsa)
      • 把客户端的公钥传递给服务器,服务器使用该公钥给客户端授权
      • 客户端可以直接免密登录

远程(共享)仓库

借助一个远程仓库,大家可以用来共享代码、历史版本等数据,实现团队成员之间代码的同步.

远程仓库操作流程

创建远程仓库

  1. 创建一个以.git结尾的目录mkdir repo.git
  2. 进入该目录 cd repo.git
  3. 初始化共享仓库 git init –bare

这样我们就创建好了一个共享的仓库,但是现在的这个仓库仅仅只是一个空的仓库,并且不允许在这个仓库中进行任何的修改

向共享仓库中推送内容

将自己开发的项目同步到这个目录当中,其他的开发者可以共享你开发的项目了.

  1. 进去自己的项目根目录 cd demo
    • git push 远程仓库的地址 本地分支名称:远程分支名称
    • git push 远程仓库的地址 本地分支名称(如果本地和远程的名称一致)
    • git push ../repo.git master

这样demo项目就被推送到共享仓库当中了

从共享仓库中取出内容

  1. 新建一个目录example(模拟另一个开发者)
  2. 进入该目录cd ./example
    • 第一次下载代码一般使用如下的命令
    • git clone 远程仓库的地址 [项目名称]
    • git clone repo.git

通过repo.git共享仓库,我们得到了一个demo的副本 如果以后还需要下载项目代码可以通过一下代码实现

  • git pull 远程仓库的地址 远程分支名称:本地分支名称
  • git pull 远程仓库的地址 远程分支名称(如果本地和远程的名称一致)
  • 添加远程仓库别名
    • git remote add 别名 远程仓库地址
  • 删除远程仓库别名
    • git remote remove 别名
  • 查看远程仓库别名的详细信息
    • git remote show 别名
  • 查看所有别名
    • git remote

基于克隆下来的项目进行修改

  • 修改内容
  • 提交内容
  • 推送内容

这样就可以实现代码的共享,但是,现在的这个共享仓库只是存放在本地的,其他人是没有办法从我们的共享仓库共享内容的!!! 那么,如果想要实现共享,则可以将自己的共享仓库放入到一个远程的服务器中,问题就可以解决了…

远程仓库服务GitHub

如果我们熟悉服务器的话，我们完全可以将上述的步骤在我们的远程服务器上进行操作，然后再做一些登录权限的设置，就可非常完美的搭建一个共享服务器了。其实为了更好的管理我们的仓库，一些第三方机构开发出了Web版仓库管理程序，通过Web界面形式管理仓库。

GitHub基本使用

  1. 注册账号
  2. 创建共享仓库

GitHub地址格式

  • https://
    • 获取代码不需要密码
    • 提交代码需要密码
  • git@
    • 需要将自己的公钥添加到GitHub上
    • 获取与提交代码都不需要密码

命令行以及git初识1

git-01

命令行概述

  • dos(windows)
  • shell(Linux,Unix,Mac)
    • csh
    • ksh
    • bash

常用命令

  • pwd(Print Working Directory)查看当前目录
  • cd(Change Directory)切换目录,如cd .. 返回上一级目录
  • ls(List)查看当前目录下的内容,如ls-al
  • mkdir(make directory)创建目录,如mkdir blog
  • touch 创建文件,如touch index.html
  • cat 查看全部文件内容,如 cat index.html
  • less 查看文件,如more etc/password less etc/password
  • rm(remove)删除文件,如rm index.html
  • rmdir(remove directory) 删除文件夹，只能删除空文件夹，不常用
  • mv (move) 移动文件或重命名，如 mv index.html ./demo/index.html
  • cp (copy) 复制文件，cp index.html ./demo/index.html
  • tab 自动补全，连按两次会将所有匹配内容显示出来
  • > 和 >>重定向，如echo hello world! > README.md，>覆盖 >>追加
  • 管道符可以将多个命令连接使用，上一次（命令）的执行结果当成下一次（命令）的参数。
  • grep 匹配内容，一般结合管道符使用

vi常用操作

  2. 底行模式 :w 保存、:w filenme另存为
  3. 底行模式 :q 退出

  9. 命令模式 u 撤销操作 可多次使用
  10. 命令模式 dd 删除当前行
  11. 命令模式 yy 复制当前行
  12. 命令模式 p 粘贴内容

  14. 命令模式 ctrl+b 向后翻页
  15. 命令模式 i 进入编辑模式, 当前光标处插入
  16. 命令模式 a 进入编辑模式, 当前光标后插入

  18. 命令模式 o 进入编辑模式, 当前行下面插入新行
  19. 命令模式 O 进入编辑模式, 当前行上面插入新行

版本控制概述

  • 本地版本控制
  • 集中版本控制
  • 分布式版本控制

版本控制对于开发来说的主要的功能:

  1. 团队成员之间进行代码的合并
  2. 维护所有代码的历史记录

git环境安装

  • windows 安装
    • 下载地址,下载之后和普通软件的安装方法一样
  • linux 安装
    • CentOS发行版：sudo yum install git
    • Ubuntu发行版：sudo apt-get install git
  • Mac 安装
    • 打开Terminal直接输入git命令，会自动提示，按提示引导安装即可

git工作原理

git的三个工作区域

  • 工作目录(当前代码所有在的代码)
  • 暂存区域(工作目录的文件一旦加到暂存区就会被git管理)
    • 本地仓库(完成功能的代码会提交到这里)

git版本控制工具流程

  1. 设置用户信息
    • get config –global user.name zhangsan
    • get config –global user.email zhangsan@qq.com
  2. 初始化本地仓库
    • git init
  3. 查看仓库中所有的文件状态
    • git status
  4. 把工作目录中的文件添加到暂存区
    • git add 文件列表
    • git add *
  5. 把暂存区中的代码添加到本地仓库
    • git commit -m 备注信息
  6. 查看提交历史
  7. 撤销文件修改
  8. 恢复版本

git管理文件的四种状态

  • 未追踪untraced(没有被git管理的文件,新创建的文件) 位于工作目录
  • 已暂存staged(一般用于存储半成品)位于暂存区
  • 已提交commited(一般存储成品,一次阶段性成果会commit一次)位于本地仓库
  • 已修改modified(位于暂存区中的代码又被修改了)位于工作目录
