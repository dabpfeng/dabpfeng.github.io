gitday-01

命令行概述

  • dos(windows)
  • shell(Linux,Unix,Mac)
    • csh
    • ksh
    • bash

常用命令

  • pwd(Print Working Directory)查看当前目录
  • cd(Change Directory)切换目录,如cd .. 返回上一级目录
  • ls(List)查看当前目录下的内容,如ls-al
  • mkdir(make directory)创建目录,如mkdir blog
  • touch 创建文件,如touch index.html
  • cat 查看全部文件内容,如 cat index.html
  • less 查看文件,如more etc/password less etc/password
  • rm(remove)删除文件,如rm index.html
  • rmdir(remove directory) 删除文件夹，只能删除空文件夹，不常用
  • mv (move) 移动文件或重命名，如 mv index.html ./demo/index.html
  • cp (copy) 复制文件，cp index.html ./demo/index.html
  • tab 自动补全，连按两次会将所有匹配内容显示出来
  • > 和 >>重定向，如echo hello world! > README.md，>覆盖 >>追加
  • 管道符可以将多个命令连接使用，上一次（命令）的执行结果当成下一次（命令）的参数。
  • grep 匹配内容，一般结合管道符使用

vi编辑器

linux操作系统标配，相当于windows下的记事本。

vi编辑器的三种模式

vi编辑器提供了三种模式 命令模式、插入模式、底行模式、每种模式下所能进行的操作是不一样的

  • 3种模式的切换如下图所示

vi常用操作

  1. 打开/创建, vi 文件路径
  2. 底行模式 :w 保存、:w filenme另存为
  3. 底行模式 :q 退出
  4. 底行模式 :wq 保存并退出 (write quit)
  5. 底行模式 :e! 撤销更改, 返回到上一次保存的状态
  6. 底行模式 :q! 不保存,强制退出
  7. 底行模式 :set nu 设置行号
  8. 命令模式 ZZ(大写) 保存并退出
  9. 命令模式 u 撤销操作 可多次使用
  10. 命令模式 dd 删除当前行
  11. 命令模式 yy 复制当前行
  12. 命令模式 p 粘贴内容
  13. 命令模式 ctrl+f 向前翻页
  14. 命令模式 ctrl+b 向后翻页
  15. 命令模式 i 进入编辑模式, 当前光标处插入
  16. 命令模式 a 进入编辑模式, 当前光标后插入
  17. 命令模式 A 进入编辑模式, 光标移动到行尾
  18. 命令模式 o 进入编辑模式, 当前行下面插入新行
  19. 命令模式 O 进入编辑模式, 当前行上面插入新行

版本控制概述

  • 本地版本控制
  • 集中版本控制
  • 分布式版本控制

版本控制对于开发来说的主要的功能:

  1. 团队成员之间进行代码的合并
  2. 维护所有代码的历史记录

git环境安装

  • windows 安装
    • 下载地址,下载之后和普通软件的安装方法一样
  • linux 安装
    • CentOS发行版：sudo yum install git
    • Ubuntu发行版：sudo apt-get install git
  • Mac 安装
    • 打开Terminal直接输入git命令，会自动提示，按提示引导安装即可

git工作原理

git的三个工作区域

  • 工作目录(当前代码所有在的代码)
  • 暂存区域(工作目录的文件一旦加到暂存区就会被git管理)
    • 本地仓库(完成功能的代码会提交到这里)

git版本控制工具流程

  1. 设置用户信息
    • get config –global user.name zhangsan
    • get config –global user.email zhangsan@qq.cn
  2. 初始化本地仓库
    • git init
  3. 查看仓库中所有的文件状态
    • git status
  4. 把工作目录中的文件添加到暂存区
    • git add 文件列表
    • git add *
  5. 把暂存区中的代码添加到本地仓库
    • git commit -m 备注信息
  6. 查看提交历史
  7. 撤销文件修改
  8. 恢复版本

git管理文件的四种状态

  • 未追踪untraced(没有被git管理的文件,新创建的文件) 位于工作目录
  • 已暂存staged(一般用于存储半成品)位于暂存区
  • 已提交commited(一般存储成品,一次阶段性成果会commit一次)位于本地仓库
  • 已修改modified(位于暂存区中的代码又被修改了)位于工作目录
